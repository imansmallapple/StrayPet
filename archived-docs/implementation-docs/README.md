## Documentation Structure

Each document typically includes:

1. **Overview** - Feature description and purpose
2. **Architecture** - System design and components
3. **Implementation Steps** - How to implement the feature
4. **API Endpoints** - Available API routes
5. **Database Schema** - Related models and fields
6. **Frontend Integration** - React/TypeScript components
7. **Testing** - Test cases and verification procedures
8. **Troubleshooting** - Common issues and solutions

## How to Use This Documentation

### For New Feature Development
1. Read the QUICK_START guide for your feature
2. Review the complete IMPLEMENTATION guide
3. Reference specific sections as needed during coding

### For Bug Fixes
1. Check DIAGNOSIS documents for known issues
2. Review VERIFICATION documents for test procedures
3. Consult implementation details if needed

### For Code Review
1. Use IMPLEMENTATION guides to understand design
2. Check test cases in verification documents
3. Validate against architecture specifications
